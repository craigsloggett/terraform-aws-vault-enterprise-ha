# Basic Vault Enterprise Deployment

This example deploys a 3-node Vault Enterprise cluster into an existing VPC
with an internal NLB. It includes a Terraform-managed root CA that signs the
Vault PKI intermediate CA certificate during the bootstrap process.

> **Warning:** The root CA in this example is generated by the Terraform `tls`
> provider and its private key is stored in Terraform state as plaintext. This
> is for demonstration and development only. For production deployments, replace
> the root CA with an external certificate authority such as AWS Private CA or
> your organization's PKI.

## Prerequisites

- Terraform >= 1.0
- An existing VPC with private and public subnets tagged
  `<vpc-name>-private-*` and `<vpc-name>-public-*`
- A Route 53 hosted zone
- An Ubuntu or Debian-based AMI
- An EC2 key pair
- A Vault Enterprise license

## Usage

1. Create a `terraform.tfvars` file with the required variables:

   ```hcl
   project_name             = "vault-dev"
   vault_enterprise_license = "02MV4U..."
   ec2_key_pair_name        = "my-key-pair"
   vpc_name                 = "my-vpc"
   route53_zone_name        = "example.com"
   ec2_ami_name             = "ubuntu-24.04-*"
   ec2_ami_owner            = "099720109477"
   ```

2. Initialize and apply:

   ```sh
   terraform init
   terraform apply
   ```

   Terraform will deploy the Vault cluster infrastructure and then wait
   (up to 30 minutes) for Vault to generate its intermediate CA CSR. Once
   the CSR appears in SSM Parameter Store, Terraform signs it with the
   demo root CA and stores the signed certificate in Secrets Manager. The
   Vault nodes pick up the signed certificate and complete the PKI
   bootstrap automatically.

3. Access the cluster through the bastion host:

   ```sh
   ssh -J ubuntu@$(terraform output -raw bastion_public_ip) ubuntu@<vault-node-ip>
   ```

## Inputs

| Name | Description | Type | Default | Required |
|------|-------------|------|---------|:--------:|
| `project_name` | Name prefix for all resources. Must be 16 characters or fewer. | `string` | n/a | yes |
| `vault_enterprise_license` | Vault Enterprise license string. | `string` | n/a | yes |
| `ec2_key_pair_name` | Name of an existing EC2 key pair for SSH access. | `string` | n/a | yes |
| `vpc_name` | Name tag of the existing VPC to deploy into. | `string` | n/a | yes |
| `route53_zone_name` | Name of the Route 53 hosted zone for the Vault DNS record. | `string` | n/a | yes |
| `ec2_ami_name` | AMI name filter for the EC2 AMI data source. | `string` | n/a | yes |
| `ec2_ami_owner` | AWS account ID that owns the AMI. | `string` | n/a | yes |
| `hcp_terraform_organization_name` | HCP Terraform organization name. Leave empty to skip JWT auth configuration. | `string` | `""` | no |

## Outputs

| Name | Description |
|------|-------------|
| `vault_url` | URL of the Vault cluster. |
| `bastion_public_ip` | Public IP of the bastion host. |
