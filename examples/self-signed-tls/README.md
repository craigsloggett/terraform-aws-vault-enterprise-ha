# Self-signed TLS Vault Enterprise Deployment

This example deploys a Vault Enterprise cluster into a new VPC with an internal
NLB. It includes a Terraform-managed root CA that signs the Vault PKI
intermediate CA certificate during the bootstrap process.

This example leverages a `terraform_data` resource to simulate the intermediate
CA signing process.

During the cloud-init process, the EC2 instances coordinate to bootstrap the
cluster automatically. Part of this process is to generate an intermediate CA
such that the private key never leaves Vault. However, since Vault exists
within a larger organization with an established root of trust, a CSR is then
generated and published to AWS during cluster bootstrap. This gives Vault the
ability to self-manage it's own PKI while using an approved CA.

The `terraform_data` resource simulating the signing process in this example
waits for the CSR to be published. Once published, Terraform then signs the
intermediate CA with the root CA generated by the TLS provider in this example.

> [!WARNING]
> The root CA in this example is generated by the Terraform `tls`
> provider and its private key is stored in Terraform state. This is for
> demonstration and development only.

## Usage

1. Copy the `defaults.auto.tfvars.example` file to `defaults.auto.tfvars` and
   populate:

```hcl
vault_enterprise_license = "LICENSE"
route53_zone_name        = "example.com"
ami_owner                = "099720109477"
ami_name                 = "ubuntu/images/hvm-ssd-gp3/ubuntu-resolute-26.04-amd64-server-20260503"
```

2. Initialize and apply:

```sh
terraform init
terraform apply
```

Terraform will deploy the Vault Enterprise cluster infrastructure and then wait
(up to 30 minutes) for Vault to generate its intermediate CA CSR. Once the CSR
appears in SSM Parameter Store, Terraform signs it with the root CA created in
this example and stores the signed certificate in Secrets Manager. The Vault
nodes pick up the signed certificate and complete the PKI bootstrap.

## Usage

### main.tf

```hcl
module "vault" {
  # tflint-ignore: terraform_module_pinned_source
  source = "git::https://github.com/craigsloggett/terraform-aws-vault-enterprise"

  vault_enterprise_license = var.vault_enterprise_license
  vault_fqdn               = var.vault_fqdn
}
```

### tls.tf

```hcl
# TLS Signing Orchestration

## Root CA

resource "tls_private_key" "root_ca" {
  algorithm   = "ECDSA"
  ecdsa_curve = "P384"
}

resource "tls_self_signed_cert" "root_ca" {
  private_key_pem = tls_private_key.root_ca.private_key_pem

  subject {
    common_name  = "Vault Root CA"
    country      = "US"
    organization = "HashiCorp Demos"
  }

  validity_period_hours = 87600
  is_ca_certificate     = true

  allowed_uses = [
    "cert_signing",
    "crl_signing",
  ]
}

## Intermediate CA Signing

data "aws_region" "this" {}

resource "terraform_data" "wait_for_csr" {
  input = module.vault.vault_pki_intermediate_ca_csr_ssm_parameter_name

  provisioner "local-exec" {
    command = "${path.module}/files/wait-for-csr.sh"
    environment = {
      PARAMETER_NAME = self.input
      TIMEOUT_SEC    = "1800"
      REGION         = data.aws_region.this.region
    }
  }
}

data "aws_ssm_parameter" "vault_pki_intermediate_ca_csr" {
  name = module.vault.vault_pki_intermediate_ca_csr_ssm_parameter_name

  depends_on = [terraform_data.wait_for_csr]
}

resource "tls_locally_signed_cert" "vault_pki_signed_intermediate_ca" {
  cert_request_pem   = data.aws_ssm_parameter.vault_pki_intermediate_ca_csr.value
  ca_private_key_pem = tls_private_key.root_ca.private_key_pem
  ca_cert_pem        = tls_self_signed_cert.root_ca.cert_pem

  validity_period_hours = 26280
  is_ca_certificate     = true

  allowed_uses = [
    "cert_signing",
    "crl_signing",
  ]
}

resource "aws_secretsmanager_secret_version" "vault_pki_signed_intermediate_ca" {
  secret_id = module.vault.vault_pki_signed_intermediate_ca_secret_arn
  secret_string = jsonencode({
    signed_intermediate_ca_pem = tls_locally_signed_cert.vault_pki_signed_intermediate_ca.cert_pem
    ca_chain_pem               = tls_self_signed_cert.root_ca.cert_pem
  })
}
```
